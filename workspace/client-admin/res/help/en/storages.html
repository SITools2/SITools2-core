Archival storage entity provides the services and functions for the storage definition, the permissions and retrieval of data. Archival storage component is the function that allows the administrator to handle the data storage (create, edit, delete, enable, disable, authorization and filtering).The following sections explain how to handle an archival storage.


Create an archival storage


  1. Click on Create
  2. Fill in the form with
    • A name
    • A description
    • A local path depending on your operating system (eg) :
      1. Windows : "D:/myStorage"
      2. Linux : "/home/data/myStorage"
      Some variable can be used as shortcuts ({$ROOT_DIRECTORY})
    • An Url attachment (urlAttach) : must start with /, the url to access the storage will be (/sitools/datastorage/user/urlAttach) 
    • Recursive Access : if the subdirectories can be accessed or not
    • Listing, when files can be listed
    • Modifiable, when files can be modified
  1. Click on Ok

NB : the protocol "file:///" will be automatically added in front of your local path


Edit an archival storage


  1. Select the archival storage item you want to edit.
  2. Follow the steps above and click on OK

Enable an archival storage

  1. Select the archival storage you want to enable.
  2. Click on Enable

Disable an archival storage

  1. Select the archival storage you want to disable.
  2. Click on Disable.

Delete an archival storage

  1. Select the archival storage you want to delete.
  2. Click on Delete.

    Configuring authorization

    1. Select the archival storage item you want to authorize.
    2. Disable the storage item.
    3. Click on Authorizations and see the authorizations component

      Custom filter

      In some cases, the authorization mechanism must deal with business information for which SITools2 does not have access. To handle these cases, a developer can create a plugin that the administrator will then deploy (for more details, see the developer guide).

      1. Select the archival storage item you want to authorize.
      2. Disable the storage item.
      3. Click on Custom filter and configure the plugin
